# 4주차 실습 정리

## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰


## 관찰 내용

### 원형 vs AABB
- 원형은 물체를 원 모양으로 단순화해서 충돌을 검사하는 방법. 모서리 정확도가 낮아 원형인 오브젝트에 사용하면 적합하다.
- AABB는 물체를 기울어지지 않는 직사각형으로 감싸서 충돌을 검사하는 방법. 사각형인 오브젝트에 사용하면 적합하다. 
### AABB vs OBB
- AABB는 Bounding Box가 항상 화면 축과 평행하며 물체가 회전하면 충돌 박스가 실제보다 커짐. 실제로 회전하는 돌 스프라이트가 기울어졌지만 충돌 사각형은 반듯하기에 스프라이트 전체를 감싸기 위해 크기가 커짐.
- OBB는 Bounding Box가 물체를 따라 같이 회전하여 실제 모양과 거의 일치함. 회전하는 돌 스프라이트가 회전하면 OBB도 같이 회전하기에 실제 닿았을때만 충돌이 감지됨.

## AI와의 대화에서 배운 것
- AABB 는 오브젝트가 회전할 때 회전된 이미지를 잘리지 않게 감싸는 새로운 사각형 크기를 자동으로 만들기 때문에 이미지가 회전할때마다 AABB의 크기가 커졌다 작아졌다 한다. 이를 해결하기 위해선 충돌 계산용 기준 크기와 화면에 보이는 회전 이미지를 분리해야 한다는것을 배웠다.

## 내 게임에 적용한다면
- FPS 게임의 총알이나 폭발 범위를 나타내고 감지할때는 원형 충돌이 더 적합할것이다. 감지가 매우 빠르며 각도 상관 없이 일정한 범위를 가진다.
